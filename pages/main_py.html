main.py

Purpose

This file initializes the complete robot system and acts as the integration layer for all subsystems. It creates hardware interfaces, shared variables, task objects, and the cooperative scheduler.

Main Role

main.py is where the full robot is assembled. It imports the software modules, configures the hardware, creates the Shares and Queues used for inter-task communication, and starts the runtime scheduler.

Key Responsibilities

  • Initialize PWM, motor drivers, encoders, line sensor array, and I2C/IMU hardware
  • Load stored line-sensor calibration data from line_calib.json when available
  • Create Shares and Queues for motor control, line following, bump detection, IMU data, and observer states
  • Instantiate all runtime tasks including motor, line-follow, IMU, observer-related shares, bump, course, and user interface tasks
  • Append tasks to the cooperative scheduler with defined priorities and periods
  • Run the scheduler continuously until interruption

Why This File Matters

This file matters because it transforms a collection of independent modules into one functioning robot. Without it, the individual controllers, sensors, and event handlers would never operate as one integrated system.
